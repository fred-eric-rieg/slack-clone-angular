Documentation of Channel Service

This is a high-level description of how the live-updating of channel's messages works. Depending on how the user interacts, the Channel Service is called and it will either get the data from Firestore or from Localstorage. The Channel Service will then update the Localstorage and the Channel Component will update the UI with existing or new data.

While all Channels are stored client-side as an Rxjs Observable to asynchoniously update the UI, Threads and Messages are stored as an Array and fetched as promises from Firestore. This is because Threads and Messages are not updated as often as Channels to minimize the amount of Firestore calls.
